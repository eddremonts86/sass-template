# Plantilla Base Next.js - Instrucciones de Implementación

## Resumen del Proyecto

Desarrollo de una plantilla base para aplicaciones Next.js que servirá como punto de partida para todos los proyectos futuros.

## Stack Tecnológico Requerido

### Framework Principal

- **Next.js** (última versión estable) - Framework React con SSR/SSG
- **Vite** - Bundler (configuración personalizada para Next.js)

### UI y Componentes

- **Shadcn/ui** - Sistema de componentes base
- **Tailwind CSS** - Framework de estilos
- **Compatibilidad con:**
  - https://www.shadcnblocks.com
  - https://www.shadcn.io
  - https://ui.aceternity.com

### Autenticación

- **Clerk.js** - Sistema de autenticación completo

### Gestión de Datos

- **TanStack Query** - Gestión de estado del servidor
- **TanStack Table** - Componentes de tabla avanzados
- **TanStack Form** - Gestión de formularios
- **ag-grid-react** - Tablas para grandes volúmenes de datos

### Estado Global

- **Zustand** - Gestión de estado global ligero

### Herramientas de Desarrollo

- **pnpm** - Gestor de paquetes (OBLIGATORIO)
- **TypeScript** - Tipado estático
- **ESLint** - Linting de código
- **Prettier** - Formateo de código
- **Husky** - Git hooks

### Documentación

- **Vitepress** - Generación de documentación
- **JSDoc** - Documentación de código

## Estructura de Páginas Obligatorias

### 1. Landing Page (Pública)

- Ruta: `/`
- Acceso: Público
- Características:
  - Diseño moderno y responsivo
  - Integración con tema claro/oscuro
  - Soporte multidioma
  - Call-to-actions para registro/login

### 2. Dashboard (Privado)

- Ruta: `/dashboard`
- Acceso: Requiere autenticación
- Características:
  - Panel de administración completo
  - Navegación lateral/superior
  - Widgets y métricas
  - Gestión de datos con tablas

## Funcionalidades Esenciales

### Multidioma

- **Idiomas soportados:** Danés (da), Inglés (en), Español (es)
- **Implementación:** next-intl o react-i18next
- **Archivos de traducción:** JSON organizados por idioma
- **Detección automática:** Basada en navegador/geolocalización

### Tema Claro/Oscuro

- **Toggle integrado** en header/navbar
- **Persistencia** en localStorage
- **Transiciones suaves** entre temas
- **Compatibilidad** con todos los componentes

### Store de Estado

- **Zustand store** para datos de usuario post-autenticación
- **Persistencia** de preferencias de usuario
- **Estado global** de UI (tema, idioma, sidebar)

## Configuración Técnica

### Bundler y Gestor de Paquetes

```bash
# Usar pnpm exclusivamente
pnpm create next-app@latest --typescript --tailwind --eslint
```

### Herramientas de Calidad de Código

#### ESLint

- Plugin para TypeScript
- Plugin para Tailwind CSS
- Plugin para React/Next.js
- Orden automático de imports

#### Prettier

- Configuración para TypeScript
- Integración con Tailwind CSS
- Orden automático de clases CSS

#### Git Hooks (Husky)

- **pre-commit:** ESLint + Prettier
- **pre-push:** Build exitoso + Tests
- **commit-msg:** Validación de formato de commits

### Convenciones de Código

- **Idioma:** Inglés en todo el código (excepto archivos de traducción)
- **Naming:** camelCase para variables, PascalCase para componentes
- **Estructura:** Modular y escalable
- **Comentarios:** JSDoc obligatorio
